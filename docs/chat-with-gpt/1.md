# 四、发现页的分析框架（你可以直接照这个实现）

我给你一套 **「三层发现分析架构」**，这是重点。

---

## 第一层：事实标签层（你已经有 70%）

这是 AI 从 landing page **直接可抽取的“客观事实”**

示例字段（你可直接落库）：

```json
{
  "pricing_type": "subscription / one-time / freemium",
  "price_range": "low / mid / high",
  "target_user": "developer / creator / SMB / enterprise",
  "cta_type": "demo / signup / waitlist / contact",
  "tech_complexity": 2,
  "feature_scope": "single / narrow / broad",
  "value_proposition_style": "效率 / 赚钱 / 安全 / 自动化"
}
```

⚠️ 这一层 **不要有观点**。

---

## 第二层：判断维度层（发现的核心）

这是你目前**最欠缺、但最有价值的一层**。

这一层不是从页面直接读出来，而是 AI **基于多个字段推断**。

### 关键判断维度（强烈建议你实现）

#### 1️⃣ 可模仿性（Imitability）

```text
高 / 中 / 低
```

判断依据（AI 可推理）：

* 技术复杂度
* 功能边界
* 定价是否简单
* 是否依赖数据 / 网络效应

---

#### 2️⃣ 需求清晰度（Demand Clarity）

```text
显性 / 半显性 / 隐性
```

依据：

* landing page 是否直说痛点
* 是否用“解决 X 问题”而不是“平台”
* CTA 是否直接

---

#### 3️⃣ 失败风险类型（Failure Mode）

```text
技术风险 / 市场风险 / 增长风险 / 信任风险
```

这个非常有“自媒体味道”。

---

#### 4️⃣ 独立开发者友好度

```text
友好 / 有挑战 / 不适合
```

---

#### 5️⃣ 叙事风格（Narrative Style）

```text
卖结果 / 卖过程 / 卖情绪 / 卖身份
```

这个是后面做“反差”的关键。

---

## 第三层：发现生成层（策展 + 自媒体套路）

这一层才是你「发现页」看到的东西。

### 这一层的输入是：

* 多个产品的「判断维度」

### 输出是：

* **专题**
* **合集**
* **标题**

# 五、发现专题是怎么“自动长出来”的？

下面是**非常关键的套路**，也是你问的重点。

---

## 套路一：反差型发现（最好用）

> 用两个“本不该同时出现的判断维度”做交集

### 示例

* **低技术复杂度 + 高定价**
* **定位模糊 + CTA 很直接**
* **功能很少 + 定价页很长**
* **不是 AI + landing page 全是 AI 话术**

👉 自动生成标题：

> 「这些产品技术不复杂，但卖得一点也不便宜」

---

## 套路二：Niche 聚焦型（自媒体最爱）

> 在一个小用户群体中再切一刀

### 示例

* target_user = developer
* pricing_type = subscription
* demand_clarity = 显性

生成：

> 「专门赚程序员钱的“小而美”产品」

---

## 套路三：认知纠偏型（建立信任）

> 明确告诉用户：你以为的和现实不一样

### 示例

* narrative_style = 卖未来
* feature_scope = 很窄
* failure_mode = 增长风险

标题：

> 「这些产品不是没需求，而是很难增长」

---

## 套路四：行动导向型（强转化）

> 明确回答“你现在能不能做”

### 示例

* imitability = 高
* indie_friendly = 友好
* demand_clarity = 显性

标题：

> 「如果你现在想做 SaaS，可以从这些方向开始」

---

# 六、发现页的 UI / 交互建议（结合 AI）

### 首页不是列表，而是「判断入口」

* 每个卡片只显示：

  * 专题标题
  * 1 句 AI 判断
  * 产品数量

点击后：

* AI 用 3–5 段话讲清楚这个专题
* 展示产品卡
* CTA：👉 **“问我你能不能做类似的”**
