# 策展文案撰写指南

## 文案展示位置与作用

### 1. Title（标题）
**位置**：卡片顶部，最醒目的位置
**字数限制**：
- 中文：6-12 字
- 英文：3-6 words

**作用**：
- 吸引眼球，让用户停下滚动
- 传达核心价值主张
- 激发好奇心和点击欲望

**风格要求**：
- ✅ 简洁有力，一针见血
- ✅ 制造反差或悬念
- ✅ 使用动词或形容词增强冲击力
- ✅ 避免平铺直叙
- ❌ 不要过长或啰嗦
- ❌ 不要使用专业术语
- ❌ 不要夸大或误导

**优秀案例**：
```
❌ 差：垂直市场的B2B SMB产品
✅ 好：小众垂直，月入5万

❌ 差：简单功能但是盈利的产品
✅ 好：功能极简，收入惊人

❌ 差：速度快体验好的轻量产品
✅ 好：快到飞起的极简工具
```

### 2. Description（描述）
**位置**：标题下方，2行截断（line-clamp-2）
**字数限制**：
- 中文：30-60 字
- 英文：20-40 words

**作用**：
- 解释策展逻辑和筛选标准
- 说明为什么这组产品值得关注
- 提供足够信息让用户决定是否深入

**风格要求**：
- ✅ 清晰说明筛选维度
- ✅ 突出核心价值和反差点
- ✅ 使用具体数字增强可信度
- ✅ 语言通顺，易于理解
- ❌ 不要堆砌技术术语
- ❌ 不要过于学术化
- ❌ 不要重复标题内容

**优秀案例**：
```
❌ 差：筛选市场投向为垂直、目标客户为B2B SMB、特征简单、成本低、MVP清晰、需求主动搜索的产品。
✅ 好：月收入5000+美元，创始人粉丝不到1000，专注垂直细分市场的B2B产品。打破"小众=低收入"的刻板印象。

❌ 差：挑战功能越复杂越赚钱的固有认知，筛选简单功能却实现可观营收的产品。
✅ 好：功能不超过5个，月收入却超1万美元。证明聚焦核心价值比功能堆砌更赚钱。
```

### 3. Insight（洞察）
**位置**：卡片底部，带图标的高亮区域（品牌色背景）
**字数限制**：
- 中文：15-30 字
- 英文：10-20 words

**作用**：
- 提炼核心启发，像金句一样
- 给出可执行的行动建议
- 让用户产生"原来如此"的顿悟感

**风格要求**：
- ✅ 简短有力，一句话说清
- ✅ 可执行，给出明确方向
- ✅ 有启发性，改变认知
- ✅ 像金句一样易于传播
- ❌ 不要空洞或鸡汤
- ❌ 不要过于抽象
- ❌ 不要重复描述内容

**优秀案例**：
```
❌ 差：垂直市场虽小，但聚焦精准痛点可实现高收入，无需大量粉丝支持。
✅ 好：小众不等于低收入，精准痛点胜过万千粉丝

❌ 差：聚焦核心价值而非功能堆砌，精准解决单一痛点往往比大而全更有效。
✅ 好：做好一件事，胜过做十件平庸事

❌ 差：以"少即是多"的MVP验证速度：先做可秒懂的价值演示，再决定是否加功能。
✅ 好：先让用户秒懂价值，再谈功能多少
```

## 文案撰写流程

### Step 1: 理解策展主题
- 明确筛选维度和冲突点
- 理解目标用户和使用场景
- 找到最核心的价值主张

### Step 2: 撰写标题
- 提炼最吸睛的反差或价值点
- 使用具体数字或形容词
- 测试多个版本，选最有冲击力的

### Step 3: 撰写描述
- 第一句说明筛选标准（用具体数字）
- 第二句说明核心价值或反差点
- 确保2行内能完整表达

### Step 4: 提炼洞察
- 从策展主题中提炼可执行建议
- 用一句话说清核心启发
- 确保简短有力，易于记忆

### Step 5: 双语对齐
- 英文不是直译，而是符合英文表达习惯
- 保持核心意思一致
- 英文更简洁，中文可以更生动

## 常见错误与改进

### 错误1：标题过长或平淡
```
❌ 垂直小众但高效变现的B2B产品
✅ 小众垂直，月入5万
```

### 错误2：描述堆砌术语
```
❌ 筛选market_scope=vertical、target_customer=b2b_smb、feature_complexity=simple或moderate...
✅ 月收入5000+美元，创始人粉丝不到1000，专注垂直细分市场的B2B产品
```

### 错误3：洞察空洞无物
```
❌ 垂直市场未充分开发，聚焦精准痛点可快速启动，是B2B SMB的蓝海战略
✅ 小众市场，大有可为
```

### 错误4：中英文不对齐
```
❌ 
title_zh: "垂直小众，高效变现"
title_en: "Niche Vertical, High Revenue"  # 过于直译

✅
title_zh: "小众垂直，月入5万"
title_en: "$50K/mo from Tiny Niches"  # 符合英文习惯
```

## 文案检查清单

撰写完成后，检查以下项目：

**标题**：
- [ ] 6-12字（中文）或3-6词（英文）
- [ ] 有冲击力，吸引眼球
- [ ] 避免术语和啰嗦
- [ ] 中英文都自然流畅

**描述**：
- [ ] 30-60字（中文）或20-40词（英文）
- [ ] 清晰说明筛选逻辑
- [ ] 突出核心价值和反差
- [ ] 2行内能完整表达

**洞察**：
- [ ] 15-30字（中文）或10-20词（英文）
- [ ] 简短有力，像金句
- [ ] 可执行，有启发性
- [ ] 易于记忆和传播

**整体**：
- [ ] 三者不重复，各有侧重
- [ ] 语言通顺，接地气
- [ ] 避免夸大和误导
- [ ] 双语质量都高，不是机翻

## 参考优秀案例

### 案例1：反差型策展
```python
title_zh="低粉高收，产品为王"
title_en="Low Followers, High Revenue"
description_zh="创始人粉丝不到1000，月收入却超1万美元。证明好产品不需要网红光环，专注价值才是王道。"
description_en="Founders with <1K followers earning $10K+/mo. Proof that great products don't need influencer status."
insight_zh="产品价值胜过个人IP，专注解决问题就够了"
insight_en="Product value beats personal brand"
```

### 案例2：行动型策展
```python
title_zh="周末就能上线的MVP"
title_en="Weekend-Launchable MVPs"
description_zh="启动成本低于5000美元，功能简单清晰，2-3天就能验证想法。适合想快速测试市场的独立开发者。"
description_en="<$5K to start, simple features, 2-3 days to validate. Perfect for indie hackers testing ideas quickly."
insight_zh="先上线验证，再完善功能"
insight_en="Ship first, polish later"
```

### 案例3：认知型策展
```python
title_zh="定价即战略"
title_en="Pricing as Strategy"
description_zh="同样的功能，不同的定价策略，收入差10倍。看这些产品如何通过定价创新打开市场。"
description_en="Same features, different pricing, 10x revenue. See how these products use pricing innovation to win."
insight_zh="定价不是成本加成，而是价值传递"
insight_en="Price communicates value, not cost"
```
